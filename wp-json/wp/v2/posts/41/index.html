Our task was to find an easy-to-use, reliable block-type storage for Kubernetes. Hence we benchmarked several Open Source solutions to see how they behave under various conditions and ultimately choosing to adopt LINSTOR.<\/p>

Objective<\/strong>:<\/p>

Prior to implementing a cloud virtualization system<\/a>, it was essential to establish a reliable block storage solution for the data of virtual machines as well container workloads.<\/p>

Researched:<\/strong><\/p>

In collaboration with experts, we developed specialized testing tool set. We conducted an extensive evaluation of various storage solutions, including Ceph<\/strong>, Mayastor<\/strong>, Vitastor<\/strong>, and LINSTOR.<\/strong>
The findings from our testing were publicly released and extensively discussed within the community.<\/p>

Implementation:<\/strong><\/p>

The initial implementation of the module included:<\/p>

  • Docker images for LINSTOR and all it’s components built from source code;<\/li>\n\n
  • Configuration tool;<\/li>\n\n
  • Documentation;<\/li>\n\n
  • Integrating tests;<\/li>\n\n
  • Grafana dashboard;<\/li>\n\n
  • Prometheus alerts with detailed courses of actions.<\/li><\/ul>

    Outcomes:<\/strong><\/p>

    The LINSTOR module was successfully implemented and has been operational since the stable release of Deckhouse v1.33.
    There were detailed training and knowledge transfer sessions, as well as several public presentations.<\/p>
